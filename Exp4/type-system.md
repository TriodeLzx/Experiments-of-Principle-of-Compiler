# 类型系统设计文档

本文仅说明实验四中实现中的类型定义与类型兼容规则。

## 1. 类型定义

当前实现支持以下类型：

- 基本类型：`int`、`float`、`bool`、`str`
- 数组类型：`int[]`、`float[]`、`bool[]`、`str[]`
- 多维数组类型：如 `int[][]`、`float[][]`

其中：

- `int` 表示整数
- `float` 表示浮点数
- `bool` 表示布尔值
- `str` 表示字符串
- 数组类型表示“某种元素类型的数组”

函数本身也记录类型信息，但当前实现中函数类型主要用于参数检查和返回值检查，内部表示形式为：

- `返回类型(参数类型1,参数类型2,...)`

例如：

- `int(int,int)`：表示返回 `int`，接收两个 `int` 参数的函数

## 2. 类型兼容规则

当前实现采用静态类型检查，兼容规则如下。

### 2.1 相同类型兼容

相同类型之间可以直接赋值或传参：

- `int <- int`
- `float <- float`
- `bool <- bool`
- `str <- str`
- `int[] <- int[]`
- `int[][] <- int[][]`

### 2.2 数值提升规则

允许整型提升为浮点型：

- `float <- int`

但不允许反向赋值：

- `int <- float` 不兼容

### 2.3 数组类型兼容

如果两个类型都是数组，则递归比较其元素类型。

例如：

- `float[] <- int[]` 兼容
- `int[] <- bool[]` 不兼容
- `int[][] <- int[][]` 兼容

### 2.4 运算规则

- 算术运算 `+ - * /`：操作数必须是数值类型 `int` 或 `float`
- 位运算与取模 `| ^ & << >> %`：操作数必须是 `int`
- 逻辑运算 `and or not`：操作数必须是 `bool`
- 关系运算 `< > <= >=`：操作数必须是数值类型
- 相等运算 `== !=`：相同类型可比较，`int` 与 `float` 之间也可比较

### 2.5 赋值与初始化规则

- 变量初始化表达式类型必须与声明类型兼容
- 赋值语句右侧表达式类型必须与左值类型兼容

例如：

- `int x = 1` 合法
- `float y = 1` 合法
- `int x = "abc"` 不合法

### 2.6 函数调用规则

- 实参数量必须与形参数量一致
- 每个实参类型必须与对应形参类型兼容
- `return` 表达式类型必须与函数声明返回类型兼容

### 2.7 数组访问规则

- 数组下标必须是 `int`
- 被下标访问的对象必须是数组类型

例如：

- `nums[0]` 合法
- `nums[true]` 不合法
