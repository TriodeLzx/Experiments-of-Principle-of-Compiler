# 设计决策记录

## 决策 1：借鉴Python采用缩进界定代码块而非花括号

### 背景
在程序结构的控制流表达上，C/C++ 和 Java 等语言多采用大括号 `{}` 和分号 `;` 作为语句和代码块的边界，而 Python 依靠严格的空格缩进进行界定，比 C/C++ 以及 Java 等语言编写起来更加方便，阅读起来也更加简洁明了。

### 决策
本语言决定取消分号和花括号，采用严格的缩进（`INDENT`/`DEDENT`）来界定代码块。

### 理由
本语言的定位是面向初学者的脚本语言。取消冗余符号可以大幅降低视觉噪音，强制的缩进规则还能帮助初学者从一开始就养成良好的代码排版习惯，使代码块的层次关系一目了然，降低学习认知负担。

### 后果
简化了语法，去除冗余分隔符，提高代码可读性，但是同时增加了词法分析器的实现难度。

## 决策 2：采用静态强类型而非动态类型

### 背景
Python、JavaScript 等主流的脚本语言通常采用动态类型，变量无需声明类型且可随时改变，尽管这种设计极大地方便了快速开发，但在运行中这种设计可能会导致潜在的类型错误，从而导致运行结果不符合预期。

### 决策
本语言决定采用静态强类型系统（用 `int`, `float`, `bool` 等标识符限制变量类型），要求变量声明时必须显式定义类型，且严格限制隐式类型转换（例如 `int` 不能直接转换为 `float`，必须通过 `float()` 进行强制转换）。

### 理由
通过静态强类型，编译器可以在编译的时候自动拦截绝大多数因类型不匹配导致的逻辑错误，防止在运行中途因为类型问题导致程序崩溃或者结果不符合预期。

### 后果
极大地增强了程序的健壮性和类型安全性，但是需要牺牲一定的便利性。

## 决策 3：精简核心关键字与内置函数命名

### 背景
在 Java 等语言中往往使用完整的单词或者短语来充当关键字或函数名，这增加了初学者的记忆量和拼写错误概率。

### 决策
本语言决定采用短命名规则定义核心关键字以及内部函数名，例如使用 `fn` 代替 `function`，使用 `ret` 代替 `return` 等。

### 理由
短关键字既保留了原本的语义特征（`fn` 一看即知是 function），又减少了记忆负担，对初学者而言是相当友好的。

### 后果
精简核心关键字与内置函数命名可以使初学者上手更快，减少拼写错误，但是代码可读性会有一定的降低。
