# IR 指令集设计文档

## 1. 设计目标

本实验采用线性三地址码作为中间表示。设计这套 IR 的主要目标有四点：

- 便于从 AST 直接生成
- 便于表示顺序语句、分支、循环、函数调用和数组访问
- 便于进行常量折叠、常量传播、死代码消除等优化
- 便于后续直接解释执行

## 2. 整体结构

IR 分为三层：

```text
IRProgram
  -> IRFunction
       -> IRInstruction
```

含义如下：

- `IRProgram`：表示整个程序
- `IRFunction`：表示一个函数对应的中间代码
- `IRInstruction`：表示一条具体的 IR 指令

这种分层方式清晰地对应了“程序 - 函数 - 指令”的组织关系，便于生成、优化和执行。

## 3. 指令集设计

本实验的 IR 一共包含 12 类指令。

### 3.1 NOP

空操作指令，不表示实际语义，主要用于优化阶段的占位，后续会被统一删除。

### 3.2 LABEL

标签指令，用于标记控制流位置，例如循环开始、分支结束等。

示例：

```text
while_begin_1:
```

### 3.3 ASSIGN

普通赋值指令，用于变量初始化和简单复制。

示例：

```text
  a = 3
  b = a
```

### 3.4 BINARY

二元运算指令，用于表示加减乘除、比较、逻辑运算和位运算。

示例：

```text
  __t1 = a + b
  __t2 = x < y
```

### 3.5 UNARY

一元运算指令，用于表示负号、逻辑非、按位取反等。

示例：

```text
  __t1 = -x
  __t2 = not flag
```

### 3.6 GOTO

无条件跳转指令。

示例：

```text
  goto while_begin_1
```

### 3.7 IF_FALSE

条件为假时跳转，用于表示 `if` 和循环判断。

示例：

```text
  if_false __t1 goto if_end_1
```

### 3.8 RETURN

返回指令。

示例：

```text
  ret x
```

### 3.9 CALL

函数调用指令。

示例：

```text
  __t1 = call fibonacci(n)
```

### 3.10 INDEX_LOAD

数组读取指令，用于从数组中取值。

示例：

```text
  __t1 = nums[i]
  __t2 = mat[0][1]
```

### 3.11 INDEX_STORE

数组写入指令，用于给数组元素赋值。

示例：

```text
  nums[i] = x
```

### 3.12 ARRAY_LITERAL

数组字面量构造指令。

示例：

```text
  __t1 = [1, 2, 3]
```

## 4. 操作数表示约定

### 4.1 常量

常量直接以内联形式出现在 IR 中，例如：

- 整数：`3`
- 浮点数：`10.5`
- 布尔值：`true`、`false`
- 字符串：`"abc"`

### 4.2 变量

为了避免作用域中的同名变量发生冲突，IR 中的用户变量会被重命名为唯一名字，例如：

- `x__1`
- `sum__3`

### 4.3 临时变量

复杂表达式在翻译时会拆成若干条简单指令，中间结果使用临时变量保存，例如：

- `__t1`
- `__t2`

### 4.4 标签

控制流位置使用自动生成的标签表示，例如：

- `if_end_1`
- `while_begin_2`
- `for_end_3`

## 5. 生成策略

IR 的生成采用“递归遍历 AST，再逐步降级”的方式。

### 5.1 表达式生成

表达式会尽量拆成简单三地址形式：

- 字面量直接作为常量
- 变量直接作为操作数
- 一元表达式生成 `UNARY`
- 二元表达式生成 `BINARY`
- 函数调用生成 `CALL`
- 数组访问生成 `INDEX_LOAD`

例如：

```text
a + b * c
```

会被拆成：

```text
__t1 = b * c
__t2 = a + __t1
```

### 5.2 语句生成

语句的转换规则如下：

- 变量声明和普通赋值生成 `ASSIGN`
- 返回语句生成 `RETURN`
- `if` 生成 `IF_FALSE + GOTO + LABEL`
- `while` 生成 `LABEL + IF_FALSE + GOTO`
- `for` 也会被降级为标签和跳转形式

### 5.3 数组生成

数组字面量用 `ARRAY_LITERAL` 表示，数组读写分别用：

- `INDEX_LOAD`
- `INDEX_STORE`

多维数组访问本质上仍然是逐层索引。

## 6. 控制流表示

IR 不保留高级的结构化控制流，而是统一转换成标签和跳转。

### 6.1 条件语句

`if` 的核心思想是：

1. 先计算条件
2. 若条件为假则跳转
3. 执行分支体
4. 最后跳到结束位置

### 6.2 循环语句

`while` 和 `for` 的核心思想是：

1. 循环开始位置设置标签
2. 判断条件
3. 条件不满足时跳出
4. 否则执行循环体
5. 跳回循环开始位置

因此，循环在 IR 中本质上都变成了条件跳转加回边跳转。
